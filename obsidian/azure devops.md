### 身份

#### 不同的access level : 
##### shakeholder
建立shakeholder不用錢

stackholder主要在於建立單子, 檢查進度, 提供feedback

不能存取repo跟pipeline
##### basic user
實際執行開發的人
會有repo跟pipeline權限
可以使用test plan執行測試與看報告

##### basic user + test plan
可以實際執行test plan的撰寫
(最貴)

### test plan

azure devops在撰寫手動測試計劃時很方便

可以看到整體執行進度, 快速建立bug單

#### 測試組成
- test case : 單一個功能的測試
- test suite : 相同性質的測試
- test plan : 一次程式的整個流程


建立不同stage對應的action變成一個test case
test case可以使用attachment加入圖片說明

### test & feedback套件

整合瀏覽器的截圖與feedback的套件, 可以直接把相關的bug跟單子做結合

可以簡單拉框框或是對敏感資訊做blur

### chart
報表會呈現tester提供的資料之外, 還可以提供網頁呈現時的header, time event等其他資訊
